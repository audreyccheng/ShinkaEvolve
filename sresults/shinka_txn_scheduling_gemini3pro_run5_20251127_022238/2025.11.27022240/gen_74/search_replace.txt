This edit reintroduces the Deterministic Polish phase using LPT (Longest Processing Time) ordering, which was a key component of successful prior generations. To pay for the computational cost of this O(N^2) phase:
1.  SA iterations are reduced from 6000 to 2500.
2.  The computationally expensive 'best_insert' operator is removed from the Simulated Annealing loop, as the final Polish phase performs a more systematic version of this optimization.
3.  The Polish phase is configured with up to 8 passes to ensure convergence (Gap Repair).
4.  Beam Search width is adjusted to 5.
