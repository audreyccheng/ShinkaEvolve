Implement "LPT-Sorted Reinsertion" and "Tail-Targeted Ruin" in the ILS phase.
1. In the Recreate phase, sort the removed transactions by length (LPT) before reinserting. This prioritizes placing larger, harder-to-fit transactions into the best available gaps first, improving packing density.
2. In the Ruin phase, add a specific mode (25% prob) that targets the end of the schedule ("Tail Ruin"). This helps break the critical path dependencies that define the makespan.
3. Refine Beam Search parameters (Gamma 1.8->1.0, slightly wider beam) to enhance the initial construction.
4. Keep the deterministic local search but optimize the evaluation budget usage.
