<DESCRIPTION>
I propose three targeted improvements to increase counter repair accuracy and confidence calibration without rewriting the whole algorithm:

1) Traffic-aware weights with caps and reweighting across passes in router redistribution:
   - Compute weights per allocation pass to reflect updated residuals and cap consumption.
   - Use tol_pair = max(0.02, 2.5/max(v_i, 1.0)), weight formula: w_i = 0.6*(1−conf_dir_i) + 0.25*min(2.0, pair_resid_i/tol_pair) + 0.15*(v_i/sum_v).
   - Halve w_i for tiny links (<1 Mbps).
   - Reduce w_i by 30% on later passes when the interface already consumed >70% of its sign-specific cap.
   - Add +0.1 weight boost when pair residual > 2*tol_pair.
   - Track per-router scaling magnitude and guard usage to improve confidence calibration.

2) Asymmetric, residual-proportional pair reconciliation with micro-iterations:
   - Run up to two micro-iterations per pair with early stop when residual within tolerance.
   - Move the lower-confidence endpoint more (alpha_low=0.40, alpha_high=0.20) and scale by residual/tolerance.
   - Respect per-direction 20% caps and update edit tracking.

3) Confidence calibration refinement:
   - Add penalties when router scaling is large (|1−s_bounded|>0.25) and when router guard usage is high (≥90%).
   - Add a small penalty when an interface’s relative edit exceeds ~35% (proxy for heavy cap consumption).
   - Add bonuses when pair residuals drop ≥50% and router imbalance decreases ≥30%.
   - Keep existing edit and cap-hit penalties and no-edit bonuses.

These changes aim to improve flow conservation accuracy, pair symmetry convergence, and calibrate confidence to reflect actual repair difficulty and magnitude.
