<DESCRIPTION>
I propose two targeted improvements focused on boosting confidence calibration while preserving (or improving) counter accuracy:

1) Final confidence calibration using invariant residuals:
   - After all repairs (pair-level and router-level), compute post-repair residuals:
     - Pair residuals: rel_diff(my_tx, peer_rx) and rel_diff(my_rx, peer_tx) for up links.
     - Router residuals: per-router imbalance rel_diff(sum_tx, sum_rx).
   - Map these residuals to confidence using a tolerant linear function and combine with status confidence. Apply a change penalty for large edits relative to the original. This calibrates confidence to reflect the actual repair quality, improving the confidence_calibration metric without changing repaired values.

2) Enforce "down implies zero traffic" for unpaired interfaces:
   - Interfaces that are not part of any connected pair but are marked down should have zero traffic. This correction enhances counter accuracy for such cases and improves status-consistent behavior.

The changes retain the current repair logic (which already achieves good counter accuracy) and focus on better confidence scoring plus a safe, invariant-driven fix for unpaired down interfaces. The topology is still used for router-level flow conservation and for computing router residuals in the confidence calibration step.

        denom = max(abs(a), abs(b), 1e-9)
        return abs(a - b) / denom

    def conf_from_residual(residual: float, tol: float) -> float:
        # Map residual to confidence: 1 at 0 residual, degrades linearly until ~0 near 5*tol
        denom = max(tol * 5.0, 1e-9)
        return clamp(1.0 - residual / denom)
