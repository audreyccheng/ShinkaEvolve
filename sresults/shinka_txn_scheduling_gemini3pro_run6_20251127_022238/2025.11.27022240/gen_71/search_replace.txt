<DESCRIPTION>
1. Reverted stochastic sampling weighting from Quartic (d^4) to Cubic (d^3). Analysis of previous generations suggests d^4 might be too aggressive, leading to loss of diversity and lower scores compared to the Cubic implementation in Gen 60.
2. Implemented Block Insertions in the Local Search (`refine`) phase. The algorithm now attempts to move blocks of 2 or 3 transactions within a local window (6). This helps resolve cluster-based conflicts where moving a single transaction is insufficient to escape a local optimum.
3. Tuned `LOOKAHEAD_TARGETS` to 4 to balance lookahead accuracy with computational overhead.

    2. Ruin-and-Recreate ILS:
       - Perturbation: Removes a random subset of transactions (Ruin).
       - Recreation: Re-inserts them greedily into best positions.
       - Refinement: Standard swap/insert local search.

        workload: Workload object containing transaction data
        num_seqs: Used to scale the beam width

        Tuple of (lowest makespan, corresponding schedule)

    ILS_CYCLES = 8

            'score': 0
