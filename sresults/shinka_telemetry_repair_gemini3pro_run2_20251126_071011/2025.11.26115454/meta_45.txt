It then arbitrates disputes by selecting the candidate (Self or Peer) that is mathematically closer to this flow-implied target, achieving superior counter repair accuracy (0.879).
- **Strict Physics Clamping with Asymmetric Logic:** Top performers like **Generation 40** and **Generation 44** enforce a strict hierarchy: physical impossibilities (RX > Peer TX) are clamped immediately ("Hard Repair"), while plausible discrepancies (RX < Peer TX) are treated as "Ambiguous" and subjected to flow arbitration. This distinction prevented the over-correction of valid packet loss events, a common failure mode in symmetric averaging approaches.
- **Outcome-Based Discrete Confidence:** **Generation 44** achieved high calibration (0.874) by abandoning continuous sigmoid functions in favor of discrete logic branches. Confidence scores are assigned based on the *verification source* of the final value (e.g., `0.98` if validated by Peer+Flow, `0.95` if validated by Flow only, `0.60` if "sketchy repair" without cross-validation).

- **Router Quality Metric:** The best program computes a `quality` score for each router (`1.0 - (imbalance / total_volume)`). - **Solidity Check:** The algorithm effectively uses a "solidity check" where peer data is trusted only if it aligns with the router's local flow state. This prevents "bad peers" (who are themselves imbalanced) from corrupting the local node.
